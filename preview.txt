以下はchatGPTにRAW5 Viewer/Converter (preview.html)を生成させたときのプロンプトです．






あなたは、HTML/CSS/JavaScriptだけで動く単一ファイルのWebアプリを作成する熟練フロントエンド開発者です。

このアプリは、microbitペンプロッタツールで現在使われているRAW5形式と完全互換でなければならない。
既存ツールとの互換性を絶対に壊さないこと。

【最重要】
このプロジェクトでいうRAW5は、
1点を「x,y,z」のCSV風テキストで表す形式ではない。
RAW5は、1フレーム5文字の16進文字列を並べたプレーンテキスト形式である。

【RAW5正式仕様】

1. 1フレームは必ず5文字
   形式は

   PYYXX

   である。

2. 各文字の意味
   - P : ペン状態を表す1文字
     - "1" = penUp（ペンを上げた移動）
     - "0" = penDown（ペンを下ろした描画）
   - YYXX : 座標を表す4桁の16進文字列
     - 上位2桁 YY = y座標
     - 下位2桁 XX = x座標

3. x, y は整数で 0〜180
   保存時は丸め込み済み整数とすること。

4. 1フレームの作り方
   - 座標値は ((y & 0xff) << 8) | (x & 0xff) で16bit値にまとめる
   - それを4桁の大文字16進文字列にする
   - 先頭にペン状態文字 P を付ける

   例:
   x=10, y=20, penDown の場合
   16bit値 = 0x140A
   フレーム = "0140A"

   x=10, y=20, penUp の場合
   フレーム = "1140A"

【RAW5テキスト全体の扱い】

RAW5データ全体は、上記5文字フレームを並べたプレーンテキストとする。

入力時は以下の両方を受け入れること:
1. 連結RAW5
   例:
   1140A0140A0150A...

2. 整形RAW5
   - フレームの間に空白があってもよい
   - 改行があってもよい
   - 20フレームごと改行されていてもよい

入力時は空白・改行を無視し、5文字ずつ切り出して解釈すること。

【出力互換性】

このアプリは最低でも次の2形式でRAW5を書き出せること:

1. 連結RAW5
   - フレーム間に空白なし
   - 改行なし
   例:
   1140A0140A0150A...

2. 20フレームごと改行RAW5
   - フレーム間は半角スペース区切り可
   - 20フレームごとに改行
   例:
   1140A 0140A 0150A ...
   （20個ごとに改行）

【描画データの意味】

ストロークは複数の点列からなる。
各ストロークは次のルールでRAW5化すること:

- 各ストロークの先頭点は penUp フレームで開始位置へ移動する
- その後、同じ先頭点を含めて各点を penDown フレームで並べる

つまり、
start = stroke[0]
出力:
1) penUp(start)
2) penDown(start)
3) penDown(2点目)
4) penDown(3点目)
...

【禁止事項】

- RAW5をCSV風 x,y,z で定義しないこと
- RAW5をJSON配列にしないこと
- ヘッダを付けないこと
- 独自拡張を加えないこと
- 座標順を XXYY に変更しないこと
- 小文字16進を前提にしないこと（大文字で出力）
- pen状態を true/false や U/D にしないこと

【必要な関数仕様】

アプリ内では、少なくとも次の仕様に対応すること:

- packFrame(penUp, x, y)
  -> "PYYXX" 形式の5文字文字列を返す

- unpackFrame(frame)
  -> { penUp, x, y } を返す

- parseRAW5(text)
  -> 空白・改行を無視して5文字ごとの配列を返す

- framesToRAW5(frames, withSpaces=false, perLine=0)
  -> 連結形式または整形形式で文字列化する

【目的】

このアプリで生成したRAW5は、
microbitペンプロッタツールの draw.html / PlotMix.html / PlotDoubler.html / 比較ビューア などでそのまま読み込めること。
互換性を最優先し、現行RAW5仕様を変更しないこと。
